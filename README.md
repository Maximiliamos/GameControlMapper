# Game Control Mapper

Game Control Mapper — Windows-приложение, которое преобразует клавиатуру и кнопки мыши в независимые сенсорные контакты. Оно предназначено прежде всего для мобильных игр, запущенных в оконном или безрамочном режиме на Windows, включая Tanks Blitz в Android-эмуляторе.

> Текущий статус: **публичная beta / release candidate**. Автоматические тесты проходят, но совместимость с конкретной игрой, эмулятором, конфигурацией мониторов и защитным ПО требует ручной проверки.

## Возможности

- касание, двойное касание и удержание;
- горизонтальный свайп;
- экранный джойстик с управлением `WASD`;
- обзор камеры относительным движением мыши;
- сенсорные зоны для левой, правой и боковых кнопок мыши;
- одновременная работа камеры, движения и нескольких кнопок через независимые touch-контакты;
- запись назначаемой клавиши кнопкой **Rec**, включая `Mouse4` и `Mouse5`;
- масштабирование координат профиля под клиентскую область выбранного окна;
- полупрозрачный редактор элементов поверх игры;
- создание, копирование, импорт, экспорт и безопасное удаление профилей;
- локальные резервные копии профилей и экспорт обезличенной диагностики;
- чтение сводной статистики Blitz XVM из локальных файлов Olenemer.

Приложение не перемещает системный курсор для выполнения привязок. `WASD` и кнопки сразу создают сенсорные контакты. Камера получает относительные дельты мыши, скрывает курсор и преобразует движение в безопасную сенсорную проводку внутри целевого окна.

## Быстрый старт

1. Запустите игру или Android-эмулятор и откройте нужную игру.
2. Запустите Game Control Mapper с тем же уровнем прав, что и целевое приложение.
3. Выберите игровое окно в блоке **«Целевое окно игры»**.
4. Выберите профиль. Для Tanks Blitz предусмотрена начальная схема управления под разрешение 1920×1080; координаты масштабируются под фактический размер окна.
5. Нажмите **«Редактировать поверх игры»**, чтобы проверить и переместить элементы. Правый щелчок по свободной области открывает меню добавления элемента.
6. Сохраните профиль и нажмите `F8`.
7. Нажмите `Ctrl`, чтобы включить обзор камерой. Повторный `Ctrl` возвращает обычную мышь и освобождает ЛКМ для меню игры.
8. Нажмите `F9`, чтобы полностью остановить управление и освободить все активные контакты.

### Горячие клавиши по умолчанию

| Клавиша | Действие |
|---|---|
| `F8` | запустить управление |
| `F9` | остановить управление и освободить контакты |
| `F10` | показать или скрыть оверлей |
| `F11` | открыть редактор поверх игры |
| `Ctrl` | переключить камеру и обычную мышь |

## Камера

Камера работает по следующей цепочке:

```text
физическое движение мыши → относительные dx/dy → чувствительность и сглаживание
→ накопление до следующего touch-кадра → перемещение сенсорного контакта
```

Само нажатие `Ctrl` не создаёт касания. Первый `TouchDown` появляется только после реального движения мыши. Для длинного поворота используется расширенная эллиптическая область вокруг центра игрового окна; при достижении физической границы экрана контакт безопасно переносится без пустого touch-кадра.

Горизонтальную и вертикальную чувствительность можно вводить дробными значениями, например `0,5`. Доступны инверсия осей, мёртвая зона, сглаживание, ускорение и ограничение скорости. Подробности: [спецификация камеры](docs/CAMERA_CONTROL_SPEC.md).

## Поддерживаемые действия

| Действие | Статус | Примечание |
|---|---|---|
| Windows Touch Injection | Реализовано | Windows 10/11 x64 |
| Tap, DoubleTap, Hold | Реализовано | отдельные независимые контакты |
| Swipe | Частично | только горизонтальный свайп |
| WASD-джойстик | Реализовано | не использует курсор |
| Камера мышью | Реализовано | Raw Input используется внутри камеры |
| MouseArea | Реализовано | ЛКМ, ПКМ, Mouse4 и Mouse5 |
| Мультитач | Реализовано | в пределах лимита Windows backend |
| Macro / Sequence | Отключено в beta | формат профиля сохранён, runtime не запускается |
| XInput / геймпад | Не поддерживается | polling отключён |
| ADB / прямая отправка в Android | Не поддерживается | события создаются на стороне Windows |
| Pinch и Rotation | Не поддерживается | жесты не реализованы |
| Interception / ViGEm | Не поддерживается | драйверы не используются |

XInput не поддерживается в текущей beta-версии; наличие модели настроек геймпада не означает работающий runtime.

Полная матрица: [docs/SUPPORT_MATRIX.md](docs/SUPPORT_MATRIX.md).

## Требования

- Windows 10 или Windows 11 x64;
- для сборки — .NET SDK 8.0.422 или совместимый последний patch-релиз .NET 8;
- целевая игра должна принимать Windows Touch Injection;
- приложение и игра должны работать с совместимым уровнем прав. Если игра запущена от администратора, перехват ввода может потребовать запуска mapper с тем же уровнем прав.

Драйверы Interception, ViGEm и ADB не требуются. Готовая self-contained сборка не требует отдельно установленного .NET Runtime.

## Сборка из исходников

```powershell
git clone https://github.com/Maximiliamos/GameControlMapper.git
cd GameControlMapper

dotnet restore GameControlMapper.sln
dotnet build GameControlMapper.sln -c Release --no-restore
dotnet test GameControlMapper.sln -c Release --no-build
dotnet run --project src/GameControlMapper/GameControlMapper.csproj -c Release
```

Основное приложение использует `net8.0-windows`, WPF и Win32 API. Release-сборка компилируется с предупреждениями как ошибками.

### Создание проверяемого release-архива

Скрипт выполняет clean, restore, build, тесты, self-contained publish для `win-x64`, создаёт ZIP-файлы и SHA-256:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-release.ps1 `
  -Version 1.0.0-rc.1
```

Артефакты появятся в каталоге `artifacts`.

## Хранение данных

- профили: каталог `Profiles` рядом с исполняемым файлом;
- резервная копия профиля: соседний файл `.bak`;
- журналы и crash report: `%LocalAppData%\GameControlMapper\Logs`;
- статистика Blitz XVM читается из `%LocalAppData%\Olenemer` и не изменяется приложением.

Телеметрии и сетевой отправки данных нет. Диагностический ZIP не включает содержимое профилей, историю нажатых клавиш, список процессов и посторонние заголовки окон. Подробнее: [политика конфиденциальности](docs/PRIVACY.md).

## Архитектура проекта

```text
src/GameControlMapper/                  основное WPF-приложение
src/GameControlMapper.TouchTestHarness/ безопасный стенд проверки touch-протокола
tests/GameControlMapper.Tests/          автоматические тесты
docs/                                   архитектура, ограничения и ручная приёмка
scripts/                                сборка и проверка release-артефактов
```

Основной путь события:

```text
KeyboardHook / MouseHook / Raw Input
        ↓
InputMappingEngine
        ↓
TouchEngine + ContactManager + TouchScheduler
        ↓
WindowsTouchBackend
        ↓
InjectTouchInput
```

Подробнее: [архитектура](docs/ARCHITECTURE.md).

## Ограничения и безопасность

- Программа не внедряет DLL в игру, не модифицирует память процессов и не обходит античит.
- Глобальные hooks и Windows Touch Injection могут блокироваться защитным ПО или политиками безопасности.
- Совместимость со всеми играми, эмуляторами и античитами не гарантируется.
- При потере фокуса управление намеренно останавливается и автоматически не возобновляется.
- Реальные touch-события, несколько мониторов, смешанный DPI и отрицательные координаты требуют ручной проверки на соответствующем оборудовании.
- Приложение пока не имеет установщика, цифровой подписи и автообновления.

Смотрите также [известные ограничения](docs/KNOWN_LIMITATIONS.md), [устранение неполадок](docs/TROUBLESHOOTING.md) и [ручную проверку релиза](docs/MANUAL_RELEASE_VALIDATION.md).
