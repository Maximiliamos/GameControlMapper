# Game Control Mapper

Version 1.0 RC supports Windows 10/11 x64 and the scope documented in [docs/SUPPORT_MATRIX.md](docs/SUPPORT_MATRIX.md). It has no telemetry or network transfer. See troubleshooting, privacy and known limitations before manual validation.

Windows WPF приложение для создания профилей управления клавиатурой и мышью для оконных и безрамочных игр на Windows. Преобразует ввод с клавиатуры/мыши в симуляцию сенсорных касаний (Touch Injection).

## Описание задачи и функционала

Программа предназначена для адаптации управления в мобильных играх (запущенных через эмуляторы или на Windows с сенсорным интерфейсом) под стандартную клавиатуру и мышь. Основная идея: создавать "виртуальные" сенсорные области и привязывать к ним клавиши/кнопки мыши.

### Реализованный функционал:

1. **WASD-джойстик**  
   - При нажатии W/A/S/D вычисляется направление и смещается виртуальный джойстик.  
   - Используется контакт ID 2.  
   - Отправляется TouchDown при первом нажатии, TouchUpdate при изменении направления, TouchUp при отпускании всех клавиш.  
   - Курсор мыши не перемещается.

2. **Обзор камеры (Ctrl + мышь)**  
   - Одно нажатие Ctrl включает CameraMouseLookService и гарантированно скрывает курсор; повторное нажатие возвращает свободную мышь.  
   - Используется контакт ID 1.  
   - Физические относительные дельты Raw Input накапливаются и применяются в ритме сенсорных кадров с учётом чувствительности, инверсии и сглаживания.  
   - Само нажатие Ctrl не создаёт касание. Первый TouchDown появляется только после физического движения мыши, после чего отправляются TouchUpdate.  
   - Для длинного непрерывного поворота используется безопасная эллиптическая проводка вокруг центра целевого окна.  
   - Системный курсор не используется для вычисления камеры и не перемещается программой.  
   - При повторном Ctrl отправляется TouchUp и обычное движение мыши восстанавливается.

3. **Левая кнопка мыши (огонь)**  
   - В боевом режиме при нажатии левой кнопки отправляется TouchDown в указанной зоне (по умолчанию 1585, 628 в 1920×1080).  
   - При отпускании отправляется TouchUp.  
   - В свободном режиме ЛКМ не перехватывается и доступна меню игры; курсор не перемещается программой.

4. **Правая кнопка мыши (прицел)**  
   - Аналогично левой кнопке, но контакт ID 4, зона по умолчанию 1645, 790 (1920×1080).

5. **Дополнительные возможности**  
   - Поддержка привязок Tap, DoubleTap, Hold, Swipe, WASD joystick, camera mouse-look и MouseArea. Macro/Sequence сохранены в формате профиля, но отключены в beta.
   - Масштабирование координат из разрешения профиля на текущее разрешение экрана.  
   - XInput не поддерживается в beta и не запускает polling.
   - Оверлей для визуализации привязок поверх игры.  
   - Профили хранятся в JSON.

✅ Инициализация Touch Injection выполняется единственным production-backend `WindowsTouchBackend`.

## Матрица возможностей Beta

| Функция | Статус | Ограничения | Ручная проверка |
|---|---|---|---|
| Windows Touch Injection, клавиатура/мышь, target-window coordinates, multitouch | Поддерживается | Windows 10/11 x64, лимит backend | Да |
| Camera mouse-look, WASD joystick, MouseArea, диагностика, backup профилей | Поддерживается | Проверяется на целевой игре | Да |
| XInput, Macro/Sequence, RawInput, Interception, ViGEm | Не поддерживается в Beta | Действие отклоняется без контакта/polling | Нет |
| ADB/Android, pinch, rotation | Не поддерживается в Beta | Не реализовано | Нет |
✅ Корректные флаги в CreateTouchInfo – DOWN/UPDATE/UP с нужными INRANGE/INCONTACT.  
✅ CameraMouseLookService использует относительные дельты Raw Input, не перемещая и не центрируя курсор.  
✅ Обработчики кликов мыши (Fire/Aim) в InputMappingEngine (как MouseArea).  
✅ WASD-джойстик не трогает курсор, только симулирует прикосновения.  
✅ Профиль по умолчанию уже содержит нужные зоны (Fire/Aim/Camera/Move).

## Требования

- Windows 10/11 x64  
- .NET 8 SDK для сборки

## Сборка и запуск

```powershell
dotnet restore
dotnet build
dotnet test
```

Профили сохраняются в папке `Profiles` (по умолчанию создается рядом с исполняемым файлом).

## Горячие клавиши (по умолчанию)

- `F8` – включить маппинг  
- `F9` – выключить маппинг  
- `F10` – показать/скрыть оверлей  
- `F11` – открыть редактор профилей

## Безопасность

Проект намеренно использует только стандартные механизмы ввода операционной системы. Не внедряет DLL в игры, не патчит память игр и не пытается обойти анти-чит. Поддержка оверлея предназначена для оконного и безрамочного режимов.
